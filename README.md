## 🚀 Шаг 1. Установка инструментов (один раз)

### 1.1 Установи Node.js

Node.js — это программа, которая нужна чтобы запустить сайт на твоём компьютере.

1. Зайди на https://nodejs.org
2. Скачай версию **LTS** (слева, большая зелёная кнопка)
3. Запусти установщик и нажимай **Next → Next → Install**

Чтобы проверить что установилось — открой **Терминал** (на Mac: `Spotlight → Terminal`, на Windows: `Пуск → cmd`) и введи:

```bash
node --version
```

Должно вывести что-то вроде `v20.x.x`. Если вывело — всё ок.

### 1.2 Установи Git

Git — это программа, которая нужна чтобы загрузить код на GitHub, а потом на Vercel.

- **Mac**: уже установлен, пропускай
- **Windows**: скачай с https://git-scm.com/download/win и установи

Проверить: в терминале введи `git --version`. Должно показать версию.

### 1.3 Установи редактор кода (рекомендую VS Code)

Скачай с https://code.visualstudio.com и установи. Это удобный редактор где можно смотреть и редактировать код.

## 🖥 Шаг 2. Запуск сайта локально

1. **Распакуй архив** с проектом в удобное место (например, `Documents/portfolio`)

2. **Открой терминал в папке проекта**:
   - **Mac**: зайди в папку через Finder → правый клик на папке → "Новый терминал в папке"
   - **Windows**: зайди в папку → в адресной строке проводника напиши `cmd` и нажми Enter
   - **Или через VS Code**: `File → Open Folder → выбери папку portfolio`, затем `Terminal → New Terminal`

3. **Установи зависимости** (один раз, займёт минуту-две):
   ```bash
   npm install
   ```

4. **Запусти сайт**:
   ```bash
   npm run dev
   ```

5. Открой в браузере: **http://localhost:3000**

Готово! Сайт работает у тебя на компе. Любые изменения в коде автоматически обновят страницу.

Чтобы остановить — в терминале нажми `Ctrl + C`.

## ✏️ Шаг 3. Как редактировать контент

### Все тексты и данные — в одном файле: `lib/data.ts`

Открой этот файл в VS Code и найдёшь три главных блока:

#### 1. `translations` — все тексты интерфейса (RU + EN)

```ts
hero: {
  name1: 'Максим',
  name2: 'Сорокин',
  typewriter: ['Продуктовый дизайнер', 'UX/UI дизайнер', ...],
  description: 'UX/UI-дизайнер...',
  ...
}
```

Поменять текст — просто исправь значение в кавычках.

#### 2. `projects` — массив проектов

Каждый проект выглядит так:

```ts
{
  slug: 'iywi',                    // URL: /projects/iywi
  category: 'saas',                // 'saas' | 'mobile' | 'web'
  featured: true,                  // true = большой проект наверху
  color: 'linear-gradient(...)',   // фон карточки
  year: '2025 — 2026',
  name: { ru: 'IYWI', en: 'IYWI' },
  shortDesc: { ru: '...', en: '...' },
  behanceUrl: 'https://...',
  metrics: [...],
  overview: { ru: '...', en: '...' },
  challenge: { ru: '...', en: '...' },
  solution: { ru: '...', en: '...' },
  results: { ru: [...], en: [...] }
}
```

**Чтобы добавить новый проект** — скопируй один блок и замени значения.

#### 3. `experiences` — опыт работы

Аналогично проектам, но проще.

### 🖼 Как добавить скриншоты проектов

1. Создай папку `public/projects/iywi/` (или любой другой слаг)
2. Положи туда PNG/JPG картинки
3. На странице проекта они пока отображаются как заглушка — в следующей итерации добавим галерею (или напиши мне и я помогу)

Пример: если положишь картинки в `public/projects/iywi/screen1.png`, то обращаться к ним можно через URL `/projects/iywi/screen1.png`.

## 🌍 Шаг 4. Деплой на Vercel (публикация в интернет)

### 4.1 Регистрация на GitHub и Vercel

1. **GitHub**: зарегистрируйся на https://github.com
2. **Vercel**: зайди на https://vercel.com → нажми **Sign up with GitHub** (так проще всего)

> 💡 И GitHub, и Vercel бесплатны для личных проектов и работают в России.

### 4.2 Создай репозиторий на GitHub

1. На https://github.com → нажми **+** в правом верхнем углу → **New repository**
2. Имя репозитория: `portfolio` (или любое)
3. **Не ставь** галочки на README и .gitignore
4. Нажми **Create repository**

### 4.3 Загрузи код на GitHub

Скопируй команды, которые показал GitHub (на странице твоего нового репозитория), или используй эти. В терминале, в папке проекта, выполняй по одной:

```bash
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/ТВОЙ_USERNAME/portfolio.git
git push -u origin main
```

> Замени `ТВОЙ_USERNAME` на свой логин GitHub.

При первом `git push` он попросит залогиниться — пройди по ссылке и авторизуйся.

### 4.4 Деплой на Vercel

1. Зайди на https://vercel.com/dashboard
2. Нажми **Add New → Project**
3. Найди свой репозиторий `portfolio` в списке → **Import**
4. Оставь все настройки по умолчанию (Vercel сам поймёт что это Next.js)
5. Нажми **Deploy**

Через минуту Vercel выдаст тебе ссылку типа `portfolio-abc123.vercel.app` — и сайт работает в интернете! 🎉

### 4.5 Автоматические обновления

Теперь всё просто: изменил код → сделал push в GitHub → Vercel сам всё пересоберёт и обновит сайт.

```bash
git add .
git commit -m "Обновил описание проекта"
git push
```

## 🌐 Шаг 5. Свой домен (опционально)

### Где купить домен из России

Эти сервисы работают с российскими картами:

| Сервис | Сайт | Цена/год | Что подойдёт |
|--------|------|----------|--------------|
| **Reg.ru** | reg.ru | ~200₽ (.ru), ~1500₽ (.com) | самый популярный |
| **Beget** | beget.com | ~190₽ (.ru) | просто, без воды |
| **Timeweb** | timeweb.com | похожие цены | тоже норм |

Хорошие варианты домена:
- `sorokin.design` (если свободен)
- `maksimsorokin.ru`
- `sorokin.pro`
- `sorokindesign.ru`

### Как привязать домен к Vercel

1. **Купи домен** на reg.ru (или где удобно)
2. В Vercel: открой свой проект → **Settings → Domains**
3. Введи свой домен (например `sorokin.design`) → **Add**
4. Vercel покажет тебе, что нужно прописать в DNS:
   - Обычно это **A-запись** `76.76.21.21` на корневой домен
   - И **CNAME-запись** `cname.vercel-dns.com` на `www.`
5. Возвращайся на сайт reg.ru → в панели управления доменом → **DNS-серверы / DNS-записи**
6. Пропиши записи как показал Vercel
7. Жди 5–30 минут (иногда до 24 часов), пока DNS обновится

Всё! Твой сайт доступен на своём домене, а на Vercel по-прежнему работает как бэкенд бесплатно.

## ❓ Что делать если что-то не работает

1. **Проверь версию Node.js**: `node --version` → должна быть 18+ 
2. **Удали `node_modules` и переустанови**:
   ```bash
   rm -rf node_modules
   npm install
   ```
3. **Проверь что сборка проходит**: `npm run build` — если есть ошибки, покажет где
4. **Посмотри логи на Vercel**: на странице проекта → Deployments → последний деплой → Logs
